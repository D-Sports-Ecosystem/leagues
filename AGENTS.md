# AGENTS.md

Guidance for AI coding agents (Cursor, Claude, Windsurf, Warp, GitHub Copilot, etc.) working in this repository.

## Project Overview

This is **not** an application. It is a **data-only repository** that holds:

- Official **logos** for 17 professional and amateur hockey leagues worldwide
- Brand **color palettes** (`colors.json`) per league, with light/dark mode variants
- **WCAG 2.1 contrast audits** (`accessibility_report.json`) for every color-pair combo
- Per-team rosters / metadata under `teams/`
- Python scripts that generate the audits and the accessible text colors

It exists to give the rest of the D-Sports ecosystem (`d-sports-api`, `d-sports-engage-native`, `d-sports-site`) a single source of truth for league branding.

**Most agent work here is data updates, not code.** Add a league, refresh a logo, fix a color, or re-run the audit. That's it.

For the deeper authoring rules and reasoning behind the data model, also read [`AGENT_IMPLEMENTATION_GUIDE.md`](./AGENT_IMPLEMENTATION_GUIDE.md).

## Repository Layout

```tree
leagues/
├── <Country>/<League>/
│   ├── logo.{svg,png}              # Highest-resolution official logo
│   ├── colors.json                 # Brand palette + accessible text colors
│   └── accessibility_report.json   # WCAG contrast audit for this league
├── teams/                          # Per-team rosters and metadata
├── accessibility_audit.txt         # Aggregated audit across all 17 leagues
├── accessibility_audit.py          # Generates the audit
├── add_text_colors.py              # Computes/writes accessible text colors
├── AGENT_IMPLEMENTATION_GUIDE.md   # Detailed authoring guide (read this!)
└── README.md                       # Public-facing overview
```

The 17 leagues currently tracked: NHL, AHL, CHL, OHL, WHL, QMJHL, ECHL, NCAA, SHL, DEL, KHL, LIIGA, NL, SL, Alps HL, Czech Extraliga, EIHL.

## Data Schema

### `colors.json`

```json
{
  "league": "NHL",
  "light": {
    "primary":   "#041E42",
    "secondary": "#C8102E",
    "accent":    "#00A9E0",
    "text": {
      "on_background": { "color": "#000000", "contrast_ratio": 21.0 },
      "on_primary":    { "color": "#FFFFFF", "contrast_ratio": 16.54 },
      "on_secondary":  { "color": "#FFFFFF", "contrast_ratio": 5.88 },
      "on_accent":     { "color": "#000000", "contrast_ratio": 7.75 }
    }
  },
  "dark": { /* same shape */ },
  "source": "nhl.com CSS custom properties"
}
```

Rules:

- **Surface colors** (`primary`, `secondary`, `accent`) must come from an authoritative source — official site CSS, brand guidelines PDF, or verified press kit. Always populate `source` with where you got them.
- **Text colors** are **not free-form** — they are computed by `add_text_colors.py` to be the higher-contrast option (`#FFFFFF` or `#000000`) for each surface. Do not edit by hand.
- The dark-mode background baseline is `#0D1117` (matches GitHub's dark surface).

### `accessibility_report.json`

Auto-generated by `accessibility_audit.py`. Don't hand-edit. Contains WCAG 2.1 contrast ratios for every foreground/background pair in both light and dark modes, with AA/AAA verdicts for normal text, large text, and UI components.

## Common Tasks

### Adding a new league

1. Create the directory: `<Country>/<LeagueAbbrev>/`.
2. Drop in the highest-resolution `logo.svg` (preferred) or `logo.png` you can source officially.
3. Author `colors.json` with `league`, `light.primary/secondary/accent`, `dark.primary/secondary/accent`, and `source`. Leave the `text` blocks empty.
4. Run `python3 add_text_colors.py` to fill in the accessible text colors for every surface.
5. Run `python3 accessibility_audit.py` to regenerate `accessibility_report.json` and append to `accessibility_audit.txt`.
6. Add a row to the league table in `README.md`.
7. Add a row to the audit summary table in `README.md`.
8. Commit logo + `colors.json` + `accessibility_report.json` + README updates in one PR.

### Updating a league's brand colors (re-brand)

1. Edit `<Country>/<League>/colors.json` — only the `primary`/`secondary`/`accent` values and `source`. Keep `text` empty (it'll be regenerated).
2. Re-run `add_text_colors.py` and `accessibility_audit.py`.
3. Update the audit summary table in `README.md` if the verdict tier changed.

### Re-running the accessibility audit

```bash
python3 accessibility_audit.py
```

Regenerates every `accessibility_report.json` plus the aggregated `accessibility_audit.txt`. Safe to run whenever — it's deterministic.

## Conventions

- **Logo format**: SVG strongly preferred. Fall back to the highest-resolution PNG you can source officially (transparent background, square or wide aspect).
- **Hex colors**: 6-digit uppercase, with leading `#`.
- **JSON**: 2-space indent, trailing newline, sorted keys are nice but not required.
- **Sources**: never invent brand colors. If you can't verify a color from an official source, leave the league out and open an issue.
- **License**: logos are property of their respective leagues — this repo redistributes them under fair-use for ecosystem branding work. Don't ship them in commercial creative without checking.

## What Not To Do

- **Don't add code that runs at runtime in another repo.** This is data; consumers fetch it.
- **Don't add a Node/Bun app here.** The Python scripts are the only build tooling.
- **Don't hand-edit `accessibility_report.json` or the `text:` blocks** — they're generated.
- **Don't commit unofficial / fan-made logos.** Source from the league's official press kit or website only.
